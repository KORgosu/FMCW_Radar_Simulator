## 3. 타겟 배치 방법

| 조작 | 동작 |
|------|------|
| **빈 공간 클릭** | 새 타겟 추가 |
| **타겟 드래그** | 거리(Range) · 각도(Angle) 변경 |
| **타겟 우클릭** | 타겟 삭제 |
| **타겟 클릭 후 V 슬라이더** | 타겟 속도 변경 |
| **타겟 클릭 후 RCS 콤보박스** | 타겟 크기(반사강도) 변경 |
| **`[DEMO]` 버튼** | 보행자·차·트럭 기본 3개 타겟 복원 |
| **`[CLEAR ALL]` 버튼** | 전체 타겟 삭제 |

### SCENE 뷰 읽는 법

```
         0°
    -30° │ +30°
  -60°   │   +60°
         ▲ RADAR

  ● 원의 크기 = RCS 크기 (클수록 강한 반사체)
  숫자 레이블 = "T1 40m +5.0" → 타겟 1번, 거리 40m, 속도 +5m/s
  동심원 = 50m 간격 거리 링
  방사선 = 15° 간격 각도선
```

> **속도 부호 규칙:** `+` = 레이더에 접근,  `-` = 레이더에서 멀어짐

## 4. 파라미터 패널 설명

### WAVEFORM

| 파라미터 | 의미 | 변경 시 영향 |
|---------|------|------------|
| **fc (GHz)** | 반송파 주파수 | λ 변경 → 속도 측정 범위(Vmax) 변경 |
| **B (MHz)** | 처프 대역폭 | **거리 분해능 ΔR = c/(2B)** 결정. 클수록 정밀 |
| **Tc (μs)** | 처프 지속 시간 | 길수록 beat 주파수 낮아짐, 긴 거리 측정 용이 |
| **PRF (Hz)** | 처프 반복 주파수 | **최대 비모호 속도 Vmax = λ·PRF/4** 결정 |

### SAMPLING

| 파라미터 | 의미 | 변경 시 영향 |
|---------|------|------------|
| **fs (MHz)** | ADC 샘플링 레이트 | **최대 탐지 거리 Rmax = fs·c/(4μ)** 결정 |
| **N_c** | 처프 수 | 클수록 속도 분해능 향상, 처리 시간 증가 |

### MIMO

| 파라미터 | 의미 |
|---------|------|
| **N_TX** | 송신 안테나 수 |
| **N_RX** | 수신 안테나 수 |
| N_virtual = N_TX × N_RX → 각도 분해능 결정 |

### CFAR

| 파라미터 | 의미 | 변경 시 영향 |
|---------|------|------------|
| **Pfa** | 오경보 확률 목표값 | 낮출수록 임계값 높아짐 → 탐지 까다로워짐 |
| **Guard** | 보호 셀 수 (한쪽) | 타겟 신호가 임계값 추정에 포함되지 않도록 보호 |
| **Train** | 훈련 셀 수 (한쪽) | 클수록 잡음 추정 안정, 처리 범위 증가 |

### NOISE

| 파라미터 | 의미 |
|---------|------|
| **N₀ (dB)** | 수신기 잡음 크기. 높일수록(덜 음수) 잡음 증가 → SNR 악화 |

## 5. Tab 1 — 신호 원리 해석

### 서브플롯 ①: TX Chirp 순간 주파수

```
주파수(MHz)
  B ┤ ──────────────/
    │             /
    │           /
  0 ┤──────────/────── 시간(μs)
    0          Tc
```

- **직선의 기울기 = 처프 레이트 μ = B/Tc**
- 기울기가 가파를수록 같은 거리에서 beat 주파수가 높아짐
- **B 슬라이더** 올리면 → 직선이 더 가파르게, 분해능 향상
- **Tc 슬라이더** 올리면 → 직선이 완만하게, beat 주파수 낮아짐

### 서브플롯 ②: TX vs RX 주파수 비교

```
주파수
   TX: ────────────/
   RX: ──────────/  (τ = 2R/c 만큼 오른쪽으로 밀림)
            ↕ fb
```

- **fb (beat frequency) = 2Rμ/c**
- **두 선 사이의 수직 간격 = fb** → 이 값이 거리에 비례
- 타겟이 멀수록 RX 선이 더 오른쪽으로 밀리고 fb가 커짐
- 타겟 드래그로 거리 바꾸면 fb 값이 실시간 변경됨을 확인

> **핵심 직관:** FMCW는 "도달 시간" 대신 "주파수 차이"로 거리를 측정한다.
> fb가 크다 = 타겟이 멀다.

### 서브플롯 ③: Beat Signal (시간 영역)

```
진폭
 +1 ┤ ∿∿∿∿∿∿∿∿∿∿∿∿∿ (Re)
    │
 -1 ┤
```

- **진동 빠르기 = fb** → 타겟이 멀수록 진동 빠름
- 복소 신호이므로 실수부(Re)와 허수부(Im)가 함께 표시됨
- 타겟이 여러 개면 여러 주파수가 겹친 복합 신호로 보임
- 잡음이 있으면 완벽한 정현파가 아닌 흔들리는 파형으로 보임

### 서브플롯 ④: Range Profile (Range-FFT 결과)

```
dB
 ┤      ▲        ▲
 ┤   (T1 40m) (T2 80m)
 ┤____________________ 거리(m)
```

- **피크 위치 = 타겟 거리**
- **피크 간격이 ΔR보다 가까우면** 두 타겟이 하나로 합쳐져 보임 (분해 불가)
- **ΔR = c/(2B)** 표시 확인 → B를 두 배로 늘리면 ΔR이 절반
- 피크 주변의 낮은 부분 = 사이드로브 (Window 함수가 억제)

## 6. Tab 2 — Range-Doppler Map 해석

### Beat Matrix 히트맵 (왼쪽)

```
처프 # (slow-time)
 64 ┤ ████████████████
    │ (각 행 = 1개 처프의 beat 신호)
  1 ┤ ████████████████
    └──────────────── 샘플 # (fast-time)
```

- **행 방향(위아래) = slow-time**: 처프가 반복될수록 타겟이 이동
- **열 방향(좌우) = fast-time**: 하나의 처프 안에서의 시간
- 밝은 수직 줄기 = 특정 fast-time 위치에 강한 신호 (= 특정 거리)

### Range Profile (가운데)

- Beat matrix를 처프별로 FFT한 후 평균낸 결과
- 여기서 **피크 위치 = 거리**, **피크 높이 = 신호 세기**

### Range-Doppler Map (오른쪽, 핵심)

```
속도(m/s)
+Vmax ┤
      │         ●  ← 접근하는 타겟 (+속도, 위쪽)
  0   ┤─────────────────────── (정지 타겟은 이 선 위)
      │   ●         ← 멀어지는 타겟 (-속도, 아래쪽)
-Vmax ┤
      └──────────────── 거리(m)
        0            Rmax
```

**읽는 법:**
- **밝은 점의 x 위치 = 타겟 거리**
- **밝은 점의 y 위치 = 타겟 속도** (위 = 접근, 아래 = 후퇴)
- **밝기 = 신호 세기 (RCS × 거리 감쇠)**
- 수평 방향으로 퍼진 줄기 = 사이드로브 (정상)

**자주 보이는 패턴:**
| 패턴 | 의미 |
|------|------|
| v=0 선 위에 점 | 정지 타겟 (건물, 벽) |
| 위쪽에 점 | 접근하는 타겟 |
| 아래쪽에 점 | 멀어지는 타겟 |
| 점이 v=±Vmax 경계 근처 | velocity ambiguity 주의 |
| 같은 거리에 여러 점 (수직) | 사이드로브 또는 다중 타겟 |

## 7. Tab 3 — CFAR 탐지 해석

### Range Profile + 임계값 그래프 (상단)

```
dB
 ┤      ▲ 타겟 (임계값 초과 → 탐지 ●)
 ┤ - - -│- - - - - - CFAR 임계값 (주황 점선, 잡음에 적응)
 ┤      │   잡음
 ┤______|__________________ 거리(m)
```

- **주황 점선 = CFAR 적응형 임계값**: 주변 잡음 수준을 따라 오르내림
- **빨간 점(●) = 탐지**: 신호가 임계값을 초과한 위치
- 잡음이 높은 구간에서는 임계값도 함께 올라감 → 오경보 방지

**Pfa 슬라이더 실험:**
- Pfa 낮춤 → 임계값 올라감 → 탐지 줄어듦 (놓칠 수 있음)
- Pfa 높임 → 임계값 내려감 → 탐지 늘어남 (오경보 증가)

### CFAR 윈도우 구조 다이어그램 (우하단)

```
[Train][Train][Guard][CUT][Guard][Train][Train]
  ←nt→        ←ng→   ↑   ←ng→   ←nt→
                     현재 판단 셀
```

| 셀 종류 | 역할 |
|---------|------|
| **CUT** (Cell Under Test) | 지금 탐지 여부를 판단하는 셀 |
| **Guard** | CUT 신호가 훈련 셀에 새어들지 않도록 보호 |
| **Train** | 주변 잡음 수준 추정에 사용 |

**수식:**
```
α = N_train × (Pfa^(-1/N_train) - 1)
임계값 T = α × mean(훈련 셀 파워)
신호 > T  →  탐지
```

## 8. Tab 4 — 통합 파이프라인 & Point Cloud 해석

### 파이프라인 인디케이터 (하단 바)

```
[CHIRP] → [BEAT MATRIX] → [RANGE FFT] → [DOPPLER FFT] → [CFAR] → [AoA]
```

- 밝게 표시된 단계 = 현재 완료된 처리 단계
- 모두 완료되면 Point Cloud가 생성됨

### Range-Doppler Map + 탐지 (왼쪽)

- **빨간 원(○)** = CFAR가 탐지한 셀
- Tab 2의 RDM과 동일하지만 탐지 결과가 오버레이됨

### Point Cloud — Top-Down View (오른쪽)

```
       Cross-Range(m)
  -100  0  +100
    │  ▲(Radar)  │
    │             │
  ●←─── 탐지된 타겟
    │             │
    └─────────────┘  Range(m)
```

**읽는 법:**
- **점의 위치** = (거리, 각도)를 (x, y) 좌표로 변환한 것
- **점의 색상**: 초록계열 = 접근 중, 파랑계열 = 멀어지는 중
- **숫자 레이블** = 해당 타겟의 추정 속도 (m/s)
- **`+` 기호** = 실제 타겟 위치 (ground truth, 씬에서 배치한 위치)
- **`●` 기호** = 레이더가 탐지·추정한 위치

> **`+`와 `●`의 간격** = 측정 오차.
> B가 클수록, N_virtual이 클수록 간격이 줄어드는 것을 확인할 수 있다.

## 9. 파라미터 변경 실험 가이드

### 실험 1: 거리 분해능 확인
```
목표: 두 타겟이 가까울 때 구분되는 최소 거리 확인

1. [DEMO] 버튼으로 타겟 초기화
2. 두 타겟을 같은 각도, 비슷한 거리(예: 40m, 45m)에 배치
3. Tab 2 → Range Profile 확인 → 두 피크가 보이는가?
4. B를 줄임 → ΔR 증가 → 두 피크가 합쳐짐
5. B를 늘림 → ΔR 감소 → 두 피크가 분리됨

확인 공식: ΔR = c / (2B) = 3×10⁸ / (2×150×10⁶) = 1.0m
```

### 실험 2: 속도 측정 한계 확인
```
목표: Velocity Ambiguity 현상 관찰

1. 타겟 하나를 배치, 속도 0으로 설정
2. Tab 2 → RDM에서 v=0 선 위에 점 확인
3. 속도를 서서히 올림
4. PRF를 낮춤 → Vmax 감소
5. 타겟 속도 > Vmax가 되면 점이 반대쪽(-속도)으로 접혀서 나타남

확인 공식: Vmax = λ × PRF / 4
```

### 실험 3: ADC 샘플링 조건
```
목표: fs가 부족할 때 aliasing 확인

1. 타겟을 Rmax 근처(~150m)에 배치
2. Tab 1 → Range Profile 확인 → 올바른 위치에 피크
3. fs를 줄임 → Rmax 감소
4. 타겟이 Rmax를 초과하면 → 피크가 엉뚱한 위치(aliasing)

확인 공식: Rmax = fs × c / (4μ)
```

### 실험 4: MIMO 각도 분해능
```
목표: 가상 배열 크기와 각도 분해능의 관계

1. 두 타겟을 같은 거리, 10° 간격 각도로 배치
2. Tab 4 → Point Cloud에서 두 점이 분리되는가?
3. N_TX × N_RX(N_virtual) 줄임 → 각도 분해능 악화 → 두 점이 합쳐짐
4. N_TX × N_RX 늘림 → 두 점이 다시 분리됨

확인 공식: 각도분해능 ≈ 2/N_virtual (라디안)
```

### 실험 5: 잡음과 CFAR
```
목표: SNR이 낮을 때 탐지 성능 변화

1. 타겟 RCS를 작게(보행자 0.5) 설정
2. N₀(잡음)를 올림 → SNR 악화
3. Tab 3 → 탐지 점이 사라지기 시작하는 N₀ 확인 (= 탐지 한계)
4. Pfa를 올리면 낮은 SNR에서도 탐지되지만 오경보도 증가
```

## 10. 자주 하는 실수 & 주의사항

### RDM에서 타겟이 보이지 않을 때

| 증상 | 원인 | 해결 |
|------|------|------|
| 피크가 없음 | 타겟이 Rmax 밖 | fs 올리거나 타겟을 안쪽으로 이동 |
| RDM 전체가 균일하게 밝음 | 잡음(N₀)이 너무 높음 | N₀ 낮춤 |
| 피크가 엉뚱한 위치 | Aliasing (fs 부족) | fs ≥ 2×fb_max 조건 확인 |
| 점이 v=0에 모두 몰림 | 모든 타겟 속도 = 0 | 타겟 속도 설정 |

### CFAR 탐지 결과 이상할 때

| 증상 | 원인 | 해결 |
|------|------|------|
| 탐지가 너무 많음 (오경보) | Pfa가 너무 높음 | Pfa 낮춤 (예: 1e-4 → 1e-6) |
| 탐지가 전혀 없음 | Pfa가 너무 낮거나 SNR 부족 | Pfa 올리거나 N₀ 낮춤 |
| 타겟 주변에 여러 탐지 | Guard 셀 부족 | Guard 값 올림 |
| 클러터(잡음 덩어리) 탐지 | Train 셀 부족 | Train 값 올림 |

### Point Cloud 위치가 틀릴 때

| 증상 | 원인 |
|------|------|
| 각도 추정이 부정확 | N_virtual 부족 (N_TX × N_RX 늘릴 것) |
| 거리 추정이 부정확 | B 부족 → ΔR이 커서 정밀도 낮음 |
| 탐지 개수가 실제보다 많음 | CFAR 오경보 → Pfa 낮출 것 |

## 핵심 공식 요약

```
거리 분해능     ΔR   = c / (2B)
최대 탐지 거리  Rmax = fs·c / (4μ)       단, μ = B/Tc
최대 탐지 속도  Vmax = λ·PRF / 4
속도 분해능     ΔV   = λ / (2·N_c·T_PRI)  단, T_PRI = 1/PRF
Beat 주파수    fb   = 2R·μ / c
ADC 조건       fs   ≥ 2·fb_max = 4·Rmax·μ/c
CFAR 임계값    T    = α·mean(train)        단, α = N(Pfa^(-1/N) - 1)
```
